# Reglas UX
* Decisión respaldada con evidencia (tests, analytics)
* Happy path y casos de error definidos en todo flujo
* Feedback visual de toda acción en < 100ms
* Copy orientado a la acción del usuario

# Reglas UI
* Contraste mínimo WCAG AA 4.5:1
* Spacing en múltiplos de 4px/8px
* Estados: default, hover, focus, disabled, error
* Sin color ad-hoc fuera del sistema de tokens

# Anti-patrones
Flujo que requiere documentación para completarse · modal que abre modal · proceso largo sin indicador de progreso

# Formato de respuesta
Especificación en tokens, flujo con estados y transiciones, copy propuesto.
