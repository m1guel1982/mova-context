# Fase 1 — Descubrimiento (antes de concluir)
Identificar: propósito, usuarios, propuesta de valor, arquitectura, módulos, procesos, entidades, flujo de información, integraciones, capacidades actuales. Si algo no tiene evidencia suficiente, indicarlo — no asumir.

# Fase 2 — Evaluación

1. **Resumen ejecutivo**: qué hace, qué problema resuelve, usuarios, propuesta de valor
2. **Fortalezas**: técnicas, funcionales, arquitectónicas, de producto
3. **Debilidades**: deuda técnica, diseño, mantenibilidad, escalabilidad, operación, UX, funcional
4. **Oportunidades de mejora** (formato de oportunidad abajo)
5. **Oportunidades IA**: caso de uso, beneficio, complejidad — solo si hay datos/madurez suficiente
6. **Oportunidades UX**: navegación, comprensión, productividad, accesibilidad
7. **Automatización**: situación actual → propuesta → beneficio, para tareas/validaciones/procesos manuales
8. **Diferenciadores competitivos**: solo propuestas implementables que generen valor visible
9. **Oportunidades de negocio**: hipótesis → valor esperado → complejidad
10. **Roadmap**: corto plazo (alto impacto/bajo esfuerzo) · mediano (alto impacto/esfuerzo medio) · largo (transformación estratégica)

# Formato de oportunidad (reutilizar en 4, 5, 7, 9)
Nombre · evidencia · descripción · problema que resuelve · beneficio esperado · impacto negocio · impacto usuario · complejidad técnica · esfuerzo · prioridad

# Reglas
Solo evidencia observada · sin funcionalidad inventada · priorizar lo realista y de valor para usuario/negocio · justificar cada recomendación · explicitar incertidumbre
