# Skill: Opportunity Discovery Checklist

KISS+DRY: see `../engineering/kiss-dry-core.md`.

Before making recommendations, reconstruct: system purpose, users, processes, architecture, modules, entities, information flow, and dependencies.

# Areas to Review

| Area            | What to Look For                                                                                                               |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Product         | missing/incomplete functionality, unnecessary complexity, user drop-off points, possible simplification                        |
| User            | UX friction, excessive steps, low discoverability, repetitive tasks, accessibility issues, loss of context                     |
| Business        | monetization opportunities, premium features, cost reduction, adoption, retention, productivity                                |
| AI              | classification, recommendation, copilot features, semantic search, anomaly detection, prediction — only when viable in context |
| Data            | missing metrics/dashboards/KPIs, missing event tracking, lack of traceability or observability                                 |
| Automation      | manual tasks, repetitive validation, manual synchronization, manual reporting                                                  |
| Architecture    | technical debt, coupling, duplication, bottlenecks, scalability/maintainability risks                                          |
| Differentiation | competitive advantage, hard-to-copy capabilities, barriers to entry                                                            |

# Opportunity Format

Name · Description · Evidence · Business Impact (1-5) · User Impact (1-5) · Technical Complexity (1-5) · Priority (High/Medium/Low) · Estimated Effort
