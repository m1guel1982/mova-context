# Skill: Opportunity Discovery Checklist
KISS+DRY: ver `../engineering/kiss-dry-core.md`.

Antes de recomendar, reconstruir: propósito del sistema, usuarios, procesos, arquitectura, módulos, entidades, flujo de información, dependencias.

# Áreas a revisar

| Área | Buscar |
|---|---|
| Producto | funcionalidad incompleta/ausente, exceso de complejidad, punto de abandono, simplificación posible |
| Usuario | fricción UX, exceso de pasos, baja discoverability, tarea repetitiva, accesibilidad, pérdida de contexto |
| Negocio | monetización, feature premium, reducción de costo, adopción, retención, productividad |
| IA | clasificación, recomendación, copiloto, búsqueda semántica, detección de anomalías, predicción — solo casos viables en el contexto observado |
| Datos | métrica/dashboard/KPI faltante, evento no registrado, trazabilidad u observabilidad ausente |
| Automatización | tarea manual, validación repetitiva, sincronización manual, reporte manual |
| Arquitectura | deuda técnica, acoplamiento, duplicación, cuello de botella, riesgo de escalabilidad/mantenibilidad |
| Diferenciación | ventaja competitiva, capacidad difícil de copiar, barrera de entrada |

# Formato por oportunidad
Nombre · descripción · evidencia · impacto negocio (1-5) · impacto usuario (1-5) · complejidad técnica (1-5) · prioridad (Alta/Media/Baja) · esfuerzo estimado
