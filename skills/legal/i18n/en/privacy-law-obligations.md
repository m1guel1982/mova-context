# {{LAW}} — Key Obligations

## Scope
* Applies to: any organization processing personal data of individuals in {{COUNTRY}}
* Authority: {{AUTHORITY}}

## Legal bases for processing (Art. 6 GDPR equivalent)
1. Consent (freely given, specific, informed, unambiguous)
2. Contract performance
3. Legal obligation
4. Vital interests
5. Public task
6. Legitimate interests (with proportionality test)

## Data subject rights
* Access — know what data is processed and how
* Rectification — correct inaccurate data
* Erasure — delete data when no longer necessary
* Objection — object to processing in certain cases
* Portability — receive data in structured format
* Automated decisions — not be subject to decisions based solely on profiling

## Controller obligations
* Appoint Data Protection Officer (DPO) if required
* Maintain records of processing activities
* Conduct Data Protection Impact Assessments (DPIA) for high-risk processing
* Notify data breaches to {{AUTHORITY}} within 72 hours
* Implement privacy by design and by default

## Key principles
* **Lawfulness** — processing must have a legal basis
* **Purpose limitation** — data collected for specified, explicit, legitimate purposes
* **Data minimization** — only data necessary for the purpose
* **Accuracy** — data must be accurate and up to date
* **Storage limitation** — not kept longer than necessary
* **Security** — appropriate technical and organizational measures
* **Accountability** — controller must be able to demonstrate compliance
